# 方法论：从269个直播到深度语言模型

## 概述

本项目通过**多阶段LLM蒸馏流程**，从269个视频（1+33+234+1）中提炼出常熟阿诺的语言模型。整个过程历时2个通宵，使用了3个LLM，经过严格的质量控制和人工验证。

## 数据来源

### 精心挑选的269个视频

我们从B站精选了269个视频，每个视频都有明确的用途：

#### 1️⃣ 阿诺编年史（1个视频）
- **来源**：B站UP主 [@抽象健美大使](https://space.bilibili.com/xxxx)

- **输出**：`biography.md`（人物经历时间线）

#### 3️⃣3️⃣ 经典梗视频（33个视频）
- **来源**：B站搜索"常熟阿诺"高播放量视频
- **主要UP主**：[@一起活捉盛亦陶](https://space.bilibili.com/xxxx)
- **内容**：阿诺原声经典名场面
- **时间跨度**：2019-2026年（早期到中期）
- **用途**：刻画阿诺的**过去**——经典语录、梗、错别字
- **输出**：
  - `memes.md`（40个经典梗）
  - `quotes.md`（34条原声语录）

#### 2️⃣3️⃣4️⃣ 直播录播（234个视频）
- **来源**：B站UP主 [@薄特特](https://space.bilibili.com/xxxx)

- **用途**：刻画阿诺的**现在**——语言规律和认知模式
- **输出**：
  - `language_patterns.md`（30条语言规律）
  - `cognitive_model.md`（6维认知模型）

#### 1️⃣ 理论基础（1个视频）
- **来源**：王一安、麦寇评价阿诺
- **核心观点**："蠢限制了坏"

### 数据质量保证

- **总时长**：约80小时原始视频
- **文本量**：约120万字转写文本
- **覆盖时间**：2019-2026年
- **场景多样性**：直播、比赛、日常、破防、整活等多种场景

### 时间维度设计：过去 + 现在 = 赛博未来

**设计理念**：
- **过去**（1个编年史 + 33个梗视频）：刻画阿诺的经典形象、标志性事件、核心梗
- **现在**（234个最新录播）：捕捉阿诺当前的语言习惯、思维模式、认知特征
- **赛博未来**：基于过去和现在的数据，构建能够持续生成"阿诺式"内容的语言模型

这种设计确保模型既有历史厚度（经典梗和语录），又有时代感（最新的语言规律），能够生成既"像阿诺"又"是阿诺"的内容。

## 蒸馏流程

### 阶段0：视频准备与转写

**工具**：OpenAI Whisper（large-v3模型）

```bash
# 批量转写视频
whisper video.mp4 --model large-v3 --language zh --output_format txt
```

**输出**：269个原始转写文本（raw_transcripts/）

**质量问题**：
- 繁体字混入
- 多段语录连在一起
- 人名识别错误（"诺哥"→"洛哥"）
- 专有名词错误（"常熟"→"常数"）
- 时间戳干扰

### 阶段1：文本清洗

**目标**：修正转写错误，保留阿诺原汁原味的语言特色

#### 1.1 自动清洗（234个直播录播）

**模型**：弱模型（成本低，速度快）
**脚本**：`scripts/01_clean_transcripts.py`

**清洗规则**：
- ✅ 繁体→简体
- ✅ 修正明显的语音识别错误
- ✅ 去除时间戳
- ❌ **不修正**阿诺本人的错误用词（申气、煞门、闰蜜等）
- ❌ **不修改**语序和语气词

**Prompt设计要点**：
```python
PROMPT_TEMPLATE = """
## 背景
你正在清洗常熟阿诺的直播转写稿。

## 阿诺的语言特色（必须保留）
- 错误用词：申气（生气）、煞门（厦门）、闰蜜（闺蜜）
- 口头禅：那我问你、再说我申气了
- 重复、语气词多、逻辑混乱

## 处理规则
1. 繁体→简体
2. 修正语音识别错误（"洛哥"→"诺哥"）
3. 保留阿诺的特色错误用词
4. 保留所有语气词、重复、口头禅
...
"""
```

**输出**：234个清洗后的文本（cleaned_transcripts/）

#### 1.2 人工清洗（33个梗视频）

**原因**：
- 视频数量少，文本短
- 名场面的原始文本非常重要，不能有任何错误
- 需要对照原视频确保准确性

**流程**：
1. 逐个对照原视频
2. 手动修正所有转写错误
3. 标注经典梗的上下文
4. 提取错别字和口头禅

**输出**：33个高质量文本（event_cleaned_transcripts/）

### 阶段2：深度分析（234个直播）

**目标**：从每个直播文本中提取语言规律和认知特征

**模型**：中等模型（强推理能力）
**脚本**：`scripts/02_analyze_transcripts.py`

#### 2.1 分析维度

**语言规律维度**：
- 口头禅使用模式（频率、位置、功能）
- 自相矛盾模式（类型、间隔、触发条件）
- 答非所问模式（问答错位规律）
- 思维跳跃模式（话题转移触发词）
- 语气词使用（密度、组合规律）
- 新发现的语言特征

**认知特征维度**：
- 工作记忆（能记住多少前文？）
- 注意力（持续时间、转移触发）
- 逻辑推理（因果理解、悖论识别）
- 情绪模式（破防触发词、情绪曲线）
- 社交认知（自我认知、他人视角）
- 语言生成（流畅度、校验机制）

#### 2.2 Prompt设计

**两阶段分析**：
1. **Part 1**：语言规律提取（输出JSON）
2. **Part 2**：认知特征推断（输出JSON）

**关键设计**：
- 要求提供具体例子和证据
- 要求标注频率和位置
- 要求推测认知机制
- 输出结构化JSON便于后续聚合

**输出**：234个分析报告（analysis/individual/*.json）

**质量控制**：
- 随机抽查10个报告验证分析深度
- 检查是否有具体例子支撑
- 检查是否有认知机制解释

### 阶段3：模式聚合

**目标**：从234个分析报告中提炼高频模式

**模型**：强模型（最强综合能力）
**脚本**：`scripts/03_aggregate_analysis.py`

#### 3.1 聚合任务

**任务一：语言规律总结**
- 识别高频模式（出现在50%+文本中）
- 识别中频模式（10-50%）
- 识别低频但典型的模式
- 归纳每个模式的变体
- 给出典型例子（3-5个真实例子）

**任务二：认知模型构建**
- 工作记忆模型（容量、遗忘速度、影响）
- 注意力模型（持续时间、转移触发）
- 逻辑推理模型（因果理解、悖论识别）
- 情绪调节模型（触发阈值、调节策略）
- 社交认知模型（自我认知、心智理论）
- 语言生成模型（流程、校验、联想）

**输出**：
- `analysis/aggregated_patterns.md`（初步的语言规律）
- `analysis/aggregated_cognitive.md`（初步的认知模型）

### 阶段4：专项深化

#### 4.1 认知模型精炼

**模型**：强模型
**脚本**：`scripts/04_generate_cognitive_model.py`

**任务**：
- 基于聚合结果，构建完整的6维认知模型
- 每个维度包含：参数估计、运作机制、证据汇总
- 解释认知缺陷如何导致语言特征

**输出**：`references/understanding/cognitive_model.md`

#### 4.2 深层结构分析

**模型**：强模型
**脚本**：`scripts/05_generate_evil_stupid.py`

**理论框架**："蠢限制了坏"

**任务**：
- 结合认知模型和语言规律
- 分析"蠢"的结构性特征
- 分析"坏"的本能性特征
- 解释"蠢如何限制坏的射程"

**输出**：`references/understanding/evil_and_stupid.md`

#### 4.3 语言规律整理

**人工操作**：
- 从聚合结果中提取30条核心规律
- 按频率分为高/中/低三档
- 每条规律包含：频率、描述、例子、认知机制
- 去重和合并相似规律

**输出**：`references/understanding/language_patterns.md`

### 阶段5：素材库构建

#### 5.1 梗库和语录（人工+LLM）

**输入**：33个人工清洗的梗视频文本
**模型**：强模型

**任务**：
- 提取40个经典梗，每个包含完整故事
- 提取34条原声语录，保留原汁原味
- 整理24个错别字，标注正确写法和语境

**输出**：
- `references/output/memes.md`
- `references/output/quotes.md`
- `references/output/typos.md`

#### 5.2 人物背景（LLM生成）

**输入**：阿诺编年史视频文本
**模型**：强模型

**任务**：
- 总结人物经历时间线（25+大事件）
- 整理家庭背景（父母、爷爷奶奶、权力结构）

**输出**：
- `references/output/biography.md`
- `references/output/family.md`

### 阶段6：架构设计

**人工设计**：
- 将文件分为 understanding/（内功心法）和 output/（弹药库）
- understanding/ 用于让模型理解阿诺的思维方式
- output/ 用于生成具体内容
- 明确告诉模型"不要输出understanding/的分析内容"

### 阶段7：Skill整合

**人工编写**：`SKILL.md`

**内容**：
- 身份定义
- 内化指令（引用understanding/）
- 输出素材库（引用output/）
- 禁止行为（9条红线）
- 输出策略（高中低频分层，话题驱动）

## 技术栈

### 模型选择

我们使用了三个不同能力级别的大语言模型：

- **弱模型**：用于文本清洗，成本低，速度快
- **中等模型**：用于深度分析，推理能力强
- **强模型**：用于模式聚合和专项深化，综合能力最强

具体模型选择和API配置请参考 `scripts/` 目录中的脚本。

## 质量控制

### 多层验证

1. **自动验证**：
   - 检查JSON格式完整性
   - 检查必需字段是否存在
   - 检查例子数量是否充足

2. **人工抽查**：
   - 随机抽查10%的分析报告
   - 检查分析深度和证据充分性
   - 检查是否有明显错误

3. **交叉验证**：
   - 对比不同文本的分析结果
   - 检查高频模式是否一致
   - 检查认知模型是否自洽

4. **实战测试**：
   - 用生成的Skill进行对话测试
   - 检查是否"像阿诺"
   - 根据反馈迭代优化

### 迭代优化

- **第1轮**：发现特征过载问题 → 添加输出策略
- **第2轮**：发现规律重复 → 去重和合并
- **第3轮**：发现频率不准 → 改为高中低三档
- **第4轮**：发现缺少安全边界 → 添加SAFETY.md

## 创新点

### 1. 多阶段LLM蒸馏
- 不是简单的prompt工程
- 而是系统性的知识提炼流程
- 每个阶段使用最适合的模型

### 2. 理论与实践分离
- understanding/：让模型理解"为什么"
- output/：让模型知道"怎么做"
- 避免模型输出分析性内容

### 3. 认知科学建模
- 不只是模仿"怎么说"
- 还解释"为什么这么说"
- 基于认知心理学的6维模型

### 4. 频率分层策略
- 避免特征过载
- 高中低三档自然分布
- 话题驱动模式切换

## 局限性

### 数据局限
- 缺少更多数据类型
- 主要是直播场景
- 缺少对话数据

### 模型局限
- LLM可能过度解读
- 认知模型是推测性的
- 无法完全复现真实阿诺

### 伦理考量
- 不代表真实人物
- 仅用于娱乐和学习
- 原始数据不公开
